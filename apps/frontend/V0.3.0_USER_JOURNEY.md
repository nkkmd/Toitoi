# v0.3.0 Frontend User Journey

**Status: release candidate** | **Last updated: 2026-07-15**

この文書は、v0.3.0で固定したframework-independent frontend boundaryを、利用者の操作順に説明します。

## 利用者フロー

```text
問いの詳細を読む
  -> 系統樹を見る
  -> 文脈条件で関連する問いを探す
  -> 比較結果から派生元を選ぶ
  -> relation typeを選び派生draftを作る
  -> 内容を確認・編集する
  -> 人間が承認する
  -> Canonical Eventとして公開する
  -> 更新された系統樹で派生を確認する
```

## 1. Inquiry detail

入力:

```text
GET /api/v1/inquiries/:id/detail
```

`createInquiryDetailModel()`を通し、本文、contexts、relationships、phase、trigger、provenance、parents、children、identityを表示します。

## 2. Lineage tree

入力:

```text
GET /api/v1/inquiries/:id/tree
```

`createLineageTreeModel()`と`renderLineageTree()`を利用します。

- root / branch / leafを区別する
- selected nodeを一つ維持する
- relation typeとprovenance概要を表示する
- nodeからdetailへ移動できる
- cycle / missing referenceを局所的なwarningとして扱う
- loading / empty / ready / errorを明示する

## 3. Context exploration

入力例:

```text
GET /api/v1/inquiries/query?climate_zone=cool-temperate
```

`createContextExplorationModel()`と`renderContextExploration()`を利用します。

- 選択中criteriaを表示する
- contextsの一致と相違を比較する
- inquiry detailへのlinkを提供する
- context上の関連をcanonical identity mergeと解釈しない
- 0件はempty、無効criteriaはerrorとして扱う

## 4. Derived inquiry draft

比較結果のinquiry IDを`sourceInquiryId`として`createDerivedInquiryDraft()`へ渡します。

v0.3.0のrelation type:

- `derived_from`
- `translated_from`
- `annotates`
- `reframes`
- `revises`
- `synthesizes`

派生draftは通常のInquiry Draft workflowを使います。

```text
draft -> in_review -> approved / rejected
```

## 5. Human review and publication

`publishApprovedDerivedInquiry()`は`approved` draftだけを受け入れます。

公開後のCanonical Eventには次を記録します。

- semantic lineage: `lineage[]`
- draft / publisher / author / AI / review: `meta.publication`
- replayable source: `provenance.sources[]`

approvalはtransport deliveryそのものではありません。publisherがCanonical Eventをtransport projectionへ変換し、delivery結果を別途管理します。

## 6. Updated lineage

公開eventをretrieve / re-ingest / persistence / replayした後、同じStandard API tree endpointを再取得します。派生eventがsource inquiryのchildとして現れ、relation typeとtransport provenanceを確認できることが完了条件です。
