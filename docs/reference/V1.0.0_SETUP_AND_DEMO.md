# Toitoi v1.0.0 Reference Setup and Demo

**Status: implementation candidate** | **Baseline: v0.9.0** | **Target: v1.0.0**

## Purpose

This document defines the canonical repository-clone-to-reference-environment path for Toitoi v1.0.0. It uses deterministic fixtures by default and does not require a live model, relay, PDS, or external network after dependency installation.

## Prerequisites

- Git
- Node.js compatible with the repository CI configuration
- Corepack
- pnpm as declared by the root `packageManager`
- writable temporary or persistent directories for storage, AI state, and the search index

## Clean installation

```bash
git clone https://github.com/nkkmd/Toitoi.git
cd Toitoi
corepack enable
corepack pnpm install --frozen-lockfile
corepack pnpm test
```

A successful workspace test run is the first reproducibility gate. It covers protocol, storage, API, frontend, AI, operations, transport, search, replay, and Conformance boundaries.

## Reference fixture

The connected reference scenario is stored at:

```text
fixtures/reference/v1.0.0/east-side-weed-scenario.json
```

The externally runnable Conformance input is:

```text
fixtures/reference/v1.0.0/conformance-input.json
```

The scenario starts from the observation that weed species differ only on the eastern side of a field and connects observation capture, AI candidates, human review, publication, cross-region discovery, semantic derivation, vocabulary mapping, transport delivery state, search, replay, and recovery expectations.

## Run the external Conformance Suite

```bash
corepack pnpm --filter @toitoi/conformance exec \
  toitoi-conformance \
  --input fixtures/reference/v1.0.0/conformance-input.json \
  --output /tmp/toitoi-v1-conformance-report.json \
  --pretty
```

Expected result:

- exit code `0`
- `conformanceVersion` equals `1.0.0`
- `compatibilityProfile` is `null`
- `passed` is `true`
- `totals.failed` is `0`

The v0.9.0 migration fixture remains available only through its explicit `v0.9.0` compatibility profile. It must not be represented as strict v1 input.

## Start the reference API

Create persistent directories outside the repository working tree:

```bash
mkdir -p "$HOME/.local/share/toitoi/storage"
mkdir -p "$HOME/.local/share/toitoi/ai"
mkdir -p "$HOME/.local/share/toitoi/search"
```

Start the API in deterministic development-compatible mode:

```bash
TOITOI_STORAGE_DIR="$HOME/.local/share/toitoi/storage" \
TOITOI_AI_STORAGE_DIR="$HOME/.local/share/toitoi/ai" \
TOITOI_SEARCH_INDEX_FILE="$HOME/.local/share/toitoi/search/toitoi.sqlite" \
TOITOI_PROTOCOL=nostr \
corepack pnpm --filter @toitoi/api start
```

This mode preserves Canonical persistence even when live transport credentials are unavailable. Delivery failures remain visible as skipped or quarantined state rather than invalidating the stored Canonical Event.

## Authenticated reference mode

For authority-boundary validation:

```bash
TOITOI_AUTH_REQUIRED=true \
TOITOI_STORAGE_DIR="$HOME/.local/share/toitoi/storage" \
TOITOI_AI_STORAGE_DIR="$HOME/.local/share/toitoi/ai" \
TOITOI_SEARCH_INDEX_FILE="$HOME/.local/share/toitoi/search/toitoi.sqlite" \
corepack pnpm --filter @toitoi/api start
```

Reference requests use:

- `X-Toitoi-Actor-Id`
- `X-Toitoi-Roles`
- `Idempotency-Key` for mutations

This deterministic authentication boundary is not OAuth or OIDC certification.

## Offline and mobile validation

The default frontend test suite includes the v1 reference offline Golden Path:

```bash
corepack pnpm --filter @toitoi/frontend test
```

It verifies:

- local observation persistence
- sensitive-location confirmation
- queued state while offline
- visible synchronization failure
- explicit retry
- preservation of the reference Canonical ID
- separation of annotation review and publication approval

## Recovery validation

The operations test suite verifies the v1 reference data against:

- backup manifest generation
- relative paths, sizes, and SHA-256 checks
- restore into a separate directory
- Canonical Event ID recovery
- migration plan, dry-run, and apply
- derived-index rebuild declarations
- corruption detection

Run it with:

```bash
corepack pnpm --filter @toitoi/operations test
```

## Optional local model

A live local model is optional and is not part of default deterministic CI. When enabled, it must remain behind the asynchronous provider-neutral AI boundary. Inference failure must not block observation persistence, and generated inquiry candidates must pass human annotation review before Draft promotion and independent publication approval.

## Persistent and rebuildable state

Durable state:

- raw storage
- Canonical storage
- durable queue state
- audit records
- deployment configuration and secrets handled outside repository fixtures

Rebuildable derived state:

- FTS5 search index
- context facets
- related-inquiry candidates
- metrics and health projections

Backups must prioritize durable state. Derived indexes should be rebuilt from Canonical storage after restore.
