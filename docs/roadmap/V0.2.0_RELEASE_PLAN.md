# Toitoi v0.2.0 Release Plan

**Status: release candidate** | **Target: v0.2.0** | **Last updated: 2026-07-13**

## 目的

v0.2.0 は、protocol / API contract の初期基準点である v0.1.0 から進み、Toitoi 固有の価値である「問いの一生」を再現可能な形で示す最初のリリースとする。

このリリースでは機能数を増やすことより、観察から生まれた inquiry が Canonical Event 化され、保存・検索・派生・provenance / lineage 表示まで到達する Golden Path を完成させることを優先する。

## リリーステーマ

> 畑の東側だけ雑草の種類が違うという観察から問いが生まれ、別地域の問いへ翻訳・派生され、比較可能な系譜として取得できる。

## 必須スコープ

### 1. Golden Path reference fixture

- 元となる observation / inquiry
- 別地域・別気候帯への派生 inquiry
- 比較観察へ進む inquiry
- context、relationship、lineage、provenance を含む
- ingest → persistence → replay → Standard API の経路で再現できる

### 2. 自動検証

- fixture が Canonical Event として受理される
- replay 後も event 数と lineage が維持される
- detail API から親子参照と provenance を取得できる
- tree API から三段階の系譜を取得できる
- context / relationship query で対象 inquiry を発見できる

### 3. 人間確認済み Inquiry Draft

- 公開前の編集・レビュー状態を Canonical Event から分離する
- `draft` → `in_review` → `approved` / `rejected` の状態遷移を定義する
- reviewer identity、review timestamp、decision を記録する
- `approved` 以外の draft が公開境界を越えられないようにする

### 4. Frontend 表示境界

- Standard API の detail response を framework-independent な render model へ変換する
- inquiry text、contexts、relationships、provenance、parent / child references を表示層へ渡す
- tree response を transport-independent な lineage model へ変換する
- loading / empty / ready / error の状態を明示する

### 5. Primary transport operational smoke

- Canonical Event を Nostr publish 経路へ渡し、公開eventを再ingestできる
- Canonical Event を Lingonberry publish 経路へ渡し、carrierから取得して再ingestできる
- default CI では production credential や外部networkを必要としない
- live smoke は明示的な環境変数でのみ有効化する

### 6. 利用者向け説明

- v0.2.0 の目的、必須項目、非対象を明記する
- Golden Path fixture とテストの場所を案内する
- Inquiry Draft と Canonical Event の境界を案内する
- frontend view model と Standard API の境界を案内する
- primary transport smoke の範囲とlive testの境界を案内する
- release notes と release runbook を確定する

- [x] Human-reviewed Inquiry Draft contract と publication guard がテストされる

- [x] frontend の inquiry detail / provenance / lineage render model が Golden Path でテストされる
- [x] Nostr と Lingonberry の publish / retrieve / re-ingest 経路が deterministic smoke test で検証される

- [x] release notes の Known Limitations が最新である

## v0.2.0 の非対象

- 新しい transport の追加
- embeddings / graph inference の高度化
- production-grade authentication / rate limiting
- 大規模 DB / search engine 最適化
- 完全な GUI lineage editor
- 人間確認を伴わない AI 自動公開
