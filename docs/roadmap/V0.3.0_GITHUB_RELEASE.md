# Toitoi v0.3.0 — 問いが育つ知識空間

v0.3.0は、v0.2.0で固定した「問いの一生」のGolden Pathを、問いを閲覧するreference implementationから、問いを発見・比較・派生できる最初の知識空間へ進めるリリースです。

## Highlights

### 問いの系統樹

- root / branch / leafを区別したlineage tree
- `translated_from`などのrelation type表示
- inquiry detailへの導線と選択状態
- provenance summary
- 欠損参照と循環参照への防御
- canonical IDとtransport source IDの双方を使ったrelation解決

### 文脈に基づく探索

- `climate_zone`
- `soil_type`
- `farming_context`
- `crop_family`

選択した条件、一致した条件、異なるcontext値を比較できます。文脈上の類似は、同一の問いであることを意味せず、canonical identityの自動統合には使用しません。

### 人間確認を伴う派生問い

次の派生関係を正式な操作対象として固定しました。

- `derived_from`
- `translated_from`
- `annotates`
- `reframes`
- `revises`
- `synthesizes`

派生問いはInquiry Draftとして作成され、`draft` → `in_review` → `approved` / `rejected`のworkflowを通ります。`approved`以外のdraftはCanonical Eventとして公開できません。

Semantic relationは`Canonical Event.lineage`へ、draft ID、publisher、author、AI関与、人間のreview結果は`meta.publication`へ記録されます。

## End-to-end validation

Default CIでは、雑草相Golden Pathを使って次の一連の流れを検証します。

```text
inquiry detail
  → lineage tree
  → context exploration
  → derived inquiry draft
  → publication guard
  → human approval
  → Canonical Event publication
  → Nostr lineage projection
  → re-ingest / persistence / replay
  → updated lineage tree
```

NostrとLingonberryのdeterministic operational smoke、Standard API、storage、replay、protocol contracts、frontend contractsもworkspace testに含まれます。

## Known limitations

- frontendはframework非依存のreference layerであり、完成したproduction GUIではありません
- production-grade authentication / authorization / rate limitingは未実装です
- embeddings必須の意味検索、graph inference、本格的なoffline synchronizationは非対象です
- default CIはdeterministic local checksであり、外部relay／carrierの可用性を保証しません
- AIによる無人公開は行わず、明示的な人間承認を必要とします

# Toitoi v0.3.0 — A knowledge space where inquiries grow

v0.3.0 turns the v0.2.0 inquiry-lifecycle Golden Path into the first integrated Toitoi knowledge-space workflow.

## Highlights

- interactive inquiry detail and lineage tree models
- context exploration and comparison across climate, soil, farming context, and crop family
- conservative identity semantics: contextual similarity never merges canonical events automatically
- reviewed derived inquiry creation with six fixed derivation types
- explicit human approval before publication
- lineage and workflow provenance preservation
- Nostr projection, re-ingest, persistence, replay, and updated lineage validation
- Lingonberry lineage and publication metadata projection validation
- cross-feature Golden Path included in default workspace CI

## Supported derivation types

- `derived_from`
- `translated_from`
- `annotates`
- `reframes`
- `revises`
- `synthesizes`

## Limitations

This remains an experimental reference release. It is not a production-grade hosted product, does not include unattended AI publication, and does not complete authentication, semantic embeddings search, graph inference, full offline synchronization, or a production frontend application.
