# Toitoi v0.3.0 Release Plan

**Status: release candidate** | **Last updated: 2026-07-15**

## Release goal

v0.3.0 advances the v0.2.0 Golden Path from a readable reference implementation into the first integrated knowledge space where inquiries can be discovered, compared, derived, reviewed, published, replayed, and returned to the lineage tree.

```text
inquiry detail
  → lineage tree
  → context exploration
  → reviewed derived inquiry
  → Canonical Event publication
  → transport projection / re-ingest / replay
  → updated lineage tree
```

## Completed mandatory scope

### Lineage tree — Issue #16

- root / branch / leaf roles
- relation type display
- node selection and inquiry detail links
- provenance summaries
- loading / empty / ready / error states
- missing-reference and cycle protection
- canonical ID and transport source ID relation matching
- keyboard-accessible and escaped HTML output

### Context exploration — Issue #17

- filtering by `climate_zone`, `soil_type`, `farming_context`, and `crop_family`
- explicit selected criteria
- matched and differing context comparison
- detail navigation
- invalid query and empty result handling
- conservative semantics: related-by-context does not mean canonical identity merge

### Reviewed derived inquiry — Issue #18

- source inquiry selection
- fixed derivation types:
  - `derived_from`
  - `translated_from`
  - `annotates`
  - `reframes`
  - `revises`
  - `synthesizes`
- Inquiry Draft and human review reuse
- publication guard for non-approved drafts
- semantic lineage in `Canonical Event.lineage`
- workflow provenance in `meta.publication`
- Nostr lineage projection
- Lingonberry lineage and publication metadata projection

### Cross-feature integration — Issue #19

- detail → lineage → context exploration → derivation → approval → publication
- Nostr projection → re-ingest → persistence → replay
- updated lineage tree and derived detail verification
- default workspace CI integration
- release notes and release runbook finalization

## Release gate

The release candidate is valid when all conditions below remain true:

- Issues #16, #17, #18, and #19 are completed
- `corepack pnpm install` succeeds
- `corepack pnpm test` succeeds
- the v0.3.0 cross-feature Golden Path succeeds
- only `approved` Inquiry Drafts can be published
- canonical identity, provenance, and replay contracts have no known regression
- Nostr and Lingonberry deterministic operational smoke tests have no known regression
- README, API, frontend, release notes, runbook, and GitHub Release content are consistent

## Non-goals

- new transports
- embeddings-required semantic search
- graph inference
- production-grade large graph optimization
- full offline synchronization
- vocabulary management UI
- free-form relation type editor
- collaborative editing
- unattended AI publication
- production authentication, authorization, or rate limiting
- completed external SDK or governance process

## Known limitations

- frontend is a framework-independent reference layer, not a completed production GUI
- deterministic CI does not guarantee external relay or carrier availability
- live transport checks require explicit environment configuration
- search remains a lightweight reference implementation
- transport-specific publication metadata may not survive every projection; semantic lineage is the stable cross-transport contract
