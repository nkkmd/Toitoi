# Toitoi v0.4.0 Release Plan

**Status: release candidate preparation** | **Started: 2026-07-16** | **Last updated: 2026-07-16**

## Release intent

v0.4.0 は、v0.3.0 で完成した human-reviewed inquiry derivation の手前に、非同期 AI 補助層を接続するリリースです。

AI は Canonical Event の正本を直接変更せず、保存済みイベントを入力として要約・タグ候補を生成し、監査可能な annotation として保持します。AI 出力の自動公開は行いません。

## Required scope

1. AI job queue
   - event ingestion と AI processing を分離する
   - duplicate job を抑止する
   - pending / processing / completed / failed を区別する
   - retry 上限とエラー記録を持つ
2. AI annotation contract
   - source event id、task、model、prompt version、raw output を保持する
   - summary / tags を schema validation できる
   - unreviewed / accepted / edited / rejected を区別する
3. Worker boundary
   - inference provider を差し替え可能にする
   - local inference を既定の想定とする
   - inference failure が ingest / persistence を停止させない
4. Persistence and recovery
   - jobs と annotations を append-only JSONL で保持する
   - 最新状態を ID 単位で復元できる
   - 中断された processing job を再起動後に再キュー化できる
5. Standard API / frontend boundary
   - canonical event と AI annotation を別の契約として返す
   - AI 生成物、人間確認状態、使用モデルを確認できる
   - annotation を通常の published inquiry と区別して表示する
6. Human-reviewed promotion
   - accepted annotation のみを Inquiry Draft の補助入力に利用できる
   - promotion 後の状態は必ず draft とする
   - v0.3.0 の publication guard を維持する
7. Golden Path
   - observation から AI job を作成する
   - deterministic provider で非同期実行する
   - unreviewed annotation を保存・表示する
   - annotation を人間が accepted にする
   - accepted annotation から Inquiry Draft を作成する
   - draft 状態での公開を拒否する
   - human review 後にのみ publication guard を通過する

## Release criteria

- AI 処理を無効化しても既存 ingest / persistence / replay / API が動作する
- AI failure が canonical event の保存を失敗させない
- annotation から入力 event、model、prompt version、raw output を追跡できる
- malformed model output を拒否できる
- retry exhausted job を明示的に識別できる
- interrupted processing job を安全に再キュー化できる
- unreviewed AI output が公開済み inquiry として扱われない
- annotation acceptance が inquiry publication approval と混同されない
- promoted draft が既存 human review と publication guard を必ず通る
- frontend で AI provenance と review state を確認できる
- v0.3.0 Golden Path と workspace tests に回帰がない
- v0.4.0 Golden Path が default CI で決定的に再現できる

## Explicitly out of scope

- AI 生成 inquiry の無人公開
- production model または実 llama.cpp provider の選定
- annotation review の production write API
- embeddings / vector database / RAG
- semantic similarity による canonical identity merge
- production-grade distributed queue
- external hosted AI service を必須とする構成
- production authentication / authorization / rate limiting
- model quality を農業上の正しさとして保証すること
