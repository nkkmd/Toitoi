# Toitoi v0.4.0 Release Runbook

**Status: draft** | **Last updated: 2026-07-16**

## Purpose

v0.4.0 の非同期AI補助層が、既存のCanonical Event、Inquiry Draft、human review、publication guardを壊さずに動作することを確認します。

## Release gate

### 1. Install and workspace regression

```bash
corepack pnpm install --frozen-lockfile
corepack pnpm test
```

期待結果:

- 全workspace testが成功する
- v0.3.0 Golden Pathに回帰がない
- v0.4.0 Golden Pathが決定的に成功する

### 2. AI queue and worker

確認項目:

- duplicate jobが抑止される
- provider failureがbounded retryされる
- retry上限到達後は`failed`になる
- inference failureがCanonical Eventの保存経路へ波及しない

### 3. Persistence and restart recovery

確認項目:

- `ai_jobs.jsonl`と`ai_annotations.jsonl`がappend-onlyで保存される
- 最新状態をID単位で復元できる
- 中断された`processing` jobが再起動時に`pending`へ戻る
- attempt countと再キュー理由が保持される

### 4. Inspection API

`TOITOI_AI_STORAGE_DIR`を設定したAPIで確認します。

```text
GET /api/v1/ai/jobs
GET /api/v1/ai/jobs/:id
GET /api/v1/ai/annotations
GET /api/v1/ai/annotations/:id
GET /api/v1/ai/events/:eventId
```

期待結果:

- job、annotation、event単位のAI情報を読み取り専用で確認できる
- `state`、`event_id`、`task`、`review_state` filterが機能する
- mutation methodは`405`になる
- AI storage未設定時も既存Standard APIが動作する

### 5. Human review boundary

確認項目:

- worker生成annotationは`unreviewed`である
- `unreviewed` annotationはInquiry Draftへ昇格できない
- `accepted` annotationから作られるものは`draft`である
- annotation acceptanceだけでは公開できない
- `draft → in_review → approved`後にだけpublication guardを通過する

### 6. Frontend contract

確認項目:

- AI要約・タグ候補が通常のinquiryと区別して表示される
- model、prompt version、review stateを確認できる
- HTML escapingが維持される
- 未確認annotationには「公開済みの問いではない」ことが表示される
- accepted annotationにも別途Draft承認が必要だと表示される

## Golden Path

`apps/frontend/test_v0_4_0_golden_path.js`で次を一続きに検証します。

1. observation由来のsummary jobをenqueueする
2. deterministic local providerで非同期処理する
3. `unreviewed` annotationを永続化する
4. frontend review modelで未公開状態を表示する
5. 人間がannotationを`accepted`にする
6. accepted annotationからInquiry Draftを作る
7. draft状態での公開を拒否する
8. human reviewで承認する
9. approved draftだけがpublication guardを通過する

## Known limitations

- production modelまたは実llama.cpp providerはこのrelease gateに含めない
- production-grade distributed queue、authentication、authorization、rate limitingは対象外
- AI品質を農業上の妥当性として保証しない
- annotation acceptance UIの書き込みAPIは対象外であり、現在のHTTP面はinspectionのみ
- AI生成物の無人公開は行わない

## Release decision

次をすべて満たした場合のみrelease candidateとします。

- default CIが成功している
- workspace regressionとv0.4.0 Golden Pathが成功している
- AI出力がCanonical Event正本を直接変更しない
- human reviewとpublication guardを迂回できない
- release plan、release notes、runbook、known limitationsが整合している
