# Toitoi v0.8.0 — Search, Vocabulary, and Reuse

## Overview

Toitoi v0.8.0 makes accumulated inquiries discoverable and reusable across words, contexts, semantic relations, provenance, review state, transports, and local vocabulary.

Search is implemented as a rebuildable SQLite FTS5 projection over Canonical Events. Canonical storage remains the source of truth. Search similarity, semantic relations, and vocabulary mappings never merge Canonical Event identity.

## Highlights

- rebuildable SQLite FTS5 search projection
- lexical search across inquiry text, observations, accepted summaries, tags, labels, contexts, relations, provenance, review state, and transport metadata
- structured filters for event type, region, climate, soil, crop, season, transport, provenance, and review state
- context, transport, and review-state facets
- explicit result classification: `exact_identity`, `explicit_relation`, and `related_candidate`
- related-inquiry exploration with outgoing relations and incoming lineage children ordered before lexical candidates
- Core, Domain, and Local Vocabulary contracts
- explicit vocabulary mapping claims that preserve locality, provenance, review state, and human-confirmation boundaries
- frontend search/context ViewModel and HTML renderer
- fixed reference dataset, deterministic ranking/filter regression, and replay-equivalence validation
- canonical `provenance.sources[]` transport/source indexing
- publication human review decision indexing

## Architectural boundaries

- Canonical storage remains authoritative.
- The FTS5 index is derived and rebuildable.
- Search failure does not block ingest, persistence, review, or publication.
- Lexical similarity and context overlap produce related candidates only.
- Semantic relations remain explicit stored edges.
- Vocabulary mappings remain explicit claims and do not silently normalize local terms.
- Existing human review and publication guards remain unchanged.
- Automatic identity merge is prohibited.

## Runtime

```bash
TOITOI_STORAGE_DIR=/path/to/storage \
TOITOI_AI_STORAGE_DIR=/path/to/ai-storage \
TOITOI_SEARCH_INDEX_FILE=/path/to/search.sqlite \
TOITOI_PROTOCOL=nostr \
corepack pnpm --filter @toitoi/api start
```

Omit `TOITOI_SEARCH_INDEX_FILE` to use an in-memory search projection.

## Known limitations

- production authentication, authorization, rate limiting, and multi-user tenancy are not implemented
- FTS5 is the lexical and structured baseline; embeddings and vector search are not included
- production RAG, graph inference, and large-scale graph visualization are out of scope
- live external transport availability is outside deterministic CI guarantees
- generated, derived, related, or mapped inquiries are not guaranteed to be agriculturally correct

## Upgrade notes

No Canonical Event migration is required. The search database is derived data and can be removed and rebuilt from canonical storage. Existing v0.7.0 workflow, derivation, storage, replay, and transport contracts remain compatible.
