# Toitoi v0.8.0 Release Plan

**Status: release-ready** | **Baseline: v0.7.0** | **Started: 2026-07-22** | **Last updated: 2026-07-22**

## Release intent

v0.8.0 makes accumulated inquiries discoverable and reusable across words, contexts, regions, semantic relations, provenance, review state, and local vocabulary. Search is a rebuildable derived projection over Canonical Events. It is not a new source of truth and never changes canonical identity.

## Implemented scope

1. SQLite FTS5 search across inquiry text, observation text, accepted summaries, tags, labels, contexts, relations, provenance, review state, and transport metadata.
2. Structured filters for event type, region, climate, soil, crop, season, transport, provenance, and review state.
3. Context facets for region, climate, soil, crop, season, transport, and review state.
4. Explicit separation of exact identity, explicit semantic relation, and related candidate.
5. Core Vocabulary, Domain Vocabulary, and Local Vocabulary contracts.
6. Mapping claims connecting local terms to shared concepts without erasing local labels.
7. Related-inquiry retrieval with explicit relations ordered before lexical candidates.
8. Search and context-exploration frontend ViewModel and HTML renderer.
9. A fixed reference dataset and deterministic ranking/filter regression tests.
10. Replay-safe search index rebuilding from canonical storage.
11. Operator and release validation documentation.

## Implementation order

```text
FTS5 projection
  → structured filters
  → context / relation exploration
  → vocabulary registry and mapping claims
  → related-candidate ranking
  → search and reuse UI
  → reference dataset and regression evaluation
  → optional embedding boundary
```

Optional embeddings may only be introduced after the lexical and structured baseline. They may rank related candidates but may not merge identity.

## Architectural boundaries

- Canonical storage remains the source of truth.
- The FTS5 database is derived data and is rebuildable.
- Search failure does not block ingest, persistence, review, or publication.
- Exact identity requires a direct canonical ID match or the existing explicit identity mechanism.
- Semantic relations remain explicit stored edges.
- Similarity produces related candidates only.
- Vocabulary mappings are claims with provenance and review state, not silent normalization.
- Local vocabulary labels and locality remain preserved.
- Existing `draft → in_review → approved → published` boundaries remain mandatory.
- No new transport, persistence authority, or publication path is introduced.

## Search document projection

Each indexable Canonical Event is projected into a search document containing:

- canonical event ID
- event type
- primary text
- observation text when present
- accepted summary text when present
- tags and labels
- context keys and values
- relation types and targets
- transport and provenance identifiers
- review state
- creation timestamp

The projection stores the canonical ID as an unindexed lookup key. Search results retain that ID and expose match semantics separately.

## Vocabulary model

### Core Vocabulary

Stable cross-domain concepts required by the Toitoi contract, including common context dimensions and shared semantic concepts.

### Domain Vocabulary

Concepts maintained for a field such as agroecology, soil science, crop production, or local ecology.

### Local Vocabulary

Community, regional, dialectal, and practice-specific terms. Local terms are not rewritten into shared terms. They are connected using explicit mapping claims.

### Mapping claim invariants

- source and target remain distinct term identifiers
- relation type is explicit
- provenance and review status are retained
- accepted exact mappings require human confirmation
- mappings never create Canonical Event identity
- disputed or alternative mappings may coexist

## Implemented API

```text
GET /api/v1/search
GET /api/v1/search/contexts
GET /api/v1/inquiries/:id/related
GET /api/v1/vocabulary/terms
GET /api/v1/vocabulary/mappings
```

The existing inquiry endpoints remain compatible. The new search boundary provides ranked results, structured facets, and explicit match classification.

## Result classification

Every classified result exposes one of:

- `exact_identity`: direct canonical ID match
- `explicit_relation`: connected by a stored semantic relation
- `related_candidate`: lexical or structured-context candidate

All search, related-inquiry, and vocabulary mapping surfaces preserve the no-merge boundary. Similarity and semantic relations do not become identity.

## Acceptance criteria

- [x] FTS5 is available through a rebuildable search projection.
- [x] Inquiry, observation, accepted summary, tags, labels, contexts, relations, provenance, review, and transport metadata are searchable.
- [x] Structured filters compose deterministically.
- [x] Region, climate, soil, crop, season, transport, and review-state facets can be explored.
- [x] Search and related results preserve canonical detail IDs and expose provenance/context metadata.
- [x] Core, Domain, and Local Vocabulary scopes are represented explicitly.
- [x] Mapping claims retain local terms, locality, provenance, status, and human-confirmation boundaries.
- [x] Exact identity, explicit relation, and related candidate are distinguishable in API and UI.
- [x] Explicit relations are ordered before lexical related candidates.
- [x] Replaying the same canonical fixture rebuilds equivalent search results and facets.
- [x] A fixed reference dataset detects ranking and filtering regressions.
- [x] Search index failure remains outside canonical ingest and publication authority.
- [x] v0.7.0 and earlier workspace tests remain green.
- [x] v0.8.0 focused CI checks pass.
- [x] Final clean CI after diagnostic cleanup passes.
- [ ] PR review and merge are complete.
- [ ] Release records, tag, and GitHub Release are finalized.

## Explicitly out of scope

- automatic identity merge from lexical similarity, contexts, relations, vocabulary mappings, or embeddings
- production RAG
- graph inference
- migration to a distributed search engine
- new transport implementations
- production authentication, authorization, or rate limiting
- autonomous AI publication
