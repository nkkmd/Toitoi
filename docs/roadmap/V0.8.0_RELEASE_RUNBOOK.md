# Toitoi v0.8.0 Release Runbook

**Status: implementation** | **Baseline: v0.7.0** | **Last updated: 2026-07-22**

## Purpose

This runbook validates v0.8.0 search, vocabulary, and reuse behavior while preserving Canonical Event identity, provenance, replay, transport independence, and human review boundaries.

## Release invariants

- Canonical storage remains the source of truth.
- SQLite FTS5 is a derived, disposable, and rebuildable projection.
- Search similarity never merges Canonical Event identity.
- Explicit semantic relations remain distinct from exact identity and lexical candidates.
- Vocabulary mappings retain scope, provenance, status, and human-confirmation state.
- Local Vocabulary terms require locality and are not silently promoted to Core Vocabulary.
- Embeddings are optional future ranking inputs, not identity evidence.

## Required environment

- Node.js 24 as configured by CI
- Corepack
- pnpm 11.3.0
- clean checkout of the v0.8.0 release candidate

## 1. Install and test

```bash
corepack enable
corepack pnpm install --frozen-lockfile
corepack pnpm test
```

The complete suite must include:

- FTS5 projection and structured filter tests
- search HTTP routing tests
- search result classification tests
- Vocabulary contract and HTTP tests
- related Inquiry ordering tests
- frontend search ViewModel and renderer tests
- fixed reference dataset ranking tests
- search replay-equivalence tests
- all v0.7.0 and earlier Golden Paths

## 2. Start the API

```bash
TOITOI_STORAGE_DIR=/path/to/storage \
TOITOI_SEARCH_INDEX_FILE=/path/to/search.sqlite \
TOITOI_PROTOCOL=nostr \
corepack pnpm --filter @toitoi/api start
```

`TOITOI_SEARCH_INDEX_FILE` is optional. Without it, the server uses an in-memory FTS5 projection.

Optional runtime Vocabulary data may be supplied through the server factory as `vocabularyTerms` and `vocabularyMappings`. These datasets must already satisfy the protocol Vocabulary contracts.

## 3. Verify search endpoints

Verify:

```text
GET /api/v1/search
GET /api/v1/search/contexts
GET /api/v1/inquiries/:id/related
GET /api/v1/vocabulary/terms
GET /api/v1/vocabulary/mappings
```

Check text search across Inquiry text, observations, accepted summaries, labels, contexts, relations, provenance, review state, and transport metadata.

Check filters for region, climate zone, soil type, crop family, season, transport, provenance, and review state.

## 4. Verify result semantics

Confirm that results use only these classifications:

```text
exact_identity
explicit_relation
related_candidate
```

Every search and related-Inquiry response must preserve `identityMerged: false`. An explicit relation may rank ahead of a lexical candidate, but it must not become an identity claim.

## 5. Verify Vocabulary boundaries

Confirm support for:

```text
core
domain
local
```

Check that Local Vocabulary terms include locality. Check that mapping claims preserve source term, target term, relation, rationale, provenance, status, and human confirmation.

Only an accepted, human-confirmed `exact_match` may be presented as `mapped_equivalent`; this still does not merge Canonical Event identity.

## 6. Verify replay equivalence

Run the fixed replay-equivalence test and confirm that direct indexing and replay-derived indexing return the same:

- result IDs for fixed queries
- structured-filter results
- context facets

Rebuilding or deleting the FTS5 file must not modify Canonical storage.

## 7. Verify frontend exploration

Verify loading, empty, error, and ready states. Search cards must show classification, context, transport, provenance, and review state. The UI must state that search results, relations, and Vocabulary mappings do not automatically merge identity.

## 8. Merge gate

Before merging:

- all GitHub Actions checks are successful
- the fixed reference dataset passes without unexplained ranking changes
- replay equivalence passes
- no unresolved review thread remains
- the PR is mergeable against `main`
- release documentation matches implemented endpoints and limitations

## Rollback guidance

If the search projection is corrupt, stop the API, remove only the configured FTS5 database, and restart to rebuild from Canonical storage. Never delete or rewrite Canonical Events to repair a derived search index.
