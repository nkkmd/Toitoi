## Objective

Before v1.0.0, maintained Toitoi documentation is organized deliberately:

- major public and operational documents are bilingual, with English followed by equivalent Japanese;
- detailed technical and implementation documents are English;
- historical records retain their existing language unless they create a current misunderstanding.

This migration does not require translating every existing file.

## Non-blocking Class B normalization

Detailed technical and implementation documents should be English. Remaining Japanese passages do not block v1.0.0 unless they conflict with a Class A contract or operation.

Class B includes:

- transport-specific specifications;
- package-internal READMEs;
- adapter and converter implementation documents;
- fixture and test explanations;
- release plans, implementation records, and final-review records;
- CI and deployment internals;
- detailed API endpoint references;
- pilot internal collection and analysis templates.

New Class B text should be English. Mixed-language Class B files should be normalized when materially edited, not through unreviewed mechanical translation.

## Class C historical documents

Historical documents retain their existing language and structure. A Class C file is changed only when it is incorrectly presented as current, points to an unsafe procedure, conflicts with a current contract, or creates a material operational misunderstanding.

## Remaining sequence before release

1. add language-policy, architecture, operations, and security links to the root README;
2. add synchronization metadata to the root README without changing its established English-then-Japanese structure;
3. run CI on the exact latest PR head;
4. perform and record the final human semantic-equivalence review;
5. update `V1.0.0_RELEASE_RECORD.md` and the PR description;
6. update release-notes administration values after merge and before publication.

## Release decision rule

The documentation language gate passes when:

- all registered Class A documents exist;
- all release-blocking Class A documents contain English followed by equivalent Japanese;
- critical operational and contract literals match;
- the exact latest head passes default CI;
- a human reviewer confirms no known material divergence.

Class B or Class C translation completeness is not a v1.0.0 release condition.
